# 验证：Vercel 反代是否生效（必做）

当你完成后端部署（VPS/Render）以及 Vercel 的 Rewrites 后，请用下面流程验证：

## 1. 基础连通性

打开浏览器，访问：
- `https://你的vercel域名/api/health`

期望：
- 返回正常 JSON（或至少不是 404/5xx）

## 2. 浏览器 Network 检查（最关键）

按 F12 -> Network：
1. 点页面上的 `X 登录`
2. 完成授权回调
3. 观察以下请求是否都成功（200/302 正常）：
   - `/api/auth/twitter`
   - `/api/auth/twitter/callback`
   - `/api/auth/me`

如果这些请求失败/跳回错误页面，基本就是 Rewrites 没配或只配了 `/api/*` 没配 `/uploads/*`（见下一条）。

## 3. 爆料提交 + 证据加载

提交一次爆料（带证据文件）后，返回列表页：
1. 确认提交后能看到爆料卡片
2. 打开爆料卡片的证据图片/视频：
   - 如果证据 404：通常是缺少 `/uploads/(.*)` rewrite

## 4. 讨论评论

进入讨论广场后：
1. 发起一条评论
2. 刷新页面确认评论仍在（说明后端写入 SQLite 成功）

## 5. 投票链路（如果你开启了捐赠解锁逻辑）

投票前先完成：
1. 捐赠验证（或先把捐赠解锁机制关掉/测试模式）
2. 再投票

如果投票失败，Network 里会有对应的 `/api/votes` 错误信息（把失败的 error code/文字贴给我，我可以定位）。
